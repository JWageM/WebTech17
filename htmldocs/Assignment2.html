Question 6

Look at the following two HTML paragraphs: 

					
<p>This is an <strong>important</strong> point.</p>
<p>This is another <b>important</b> point.</p>
Explain why the HTML of the first paragraph should be preferred. Illustrate this with an example. Also, identify another pair of HTML constructs for which the same rationale holds.

<Strong> is a semantical tag, used for content markup, while <b> is used for style markup. According to this stack overflow forum:

"The tag strong conveys a meaning - showing the text strongly - while b (for bold) conveys a method - bolding the text. With strong, your code still makes sense if you use CSS stylesheets to change what the methods of making the text strong is."
Another pair of HTML constructs for which the same rationale holds are the <i> and <em> tags where both put the text in italics but <em> has a semantic meaning for emphasis whilst <i> does not.

Question 8

Describe the pros and cons of using the iframe tag in HTML

The pros and cons of iframe are decribed below.

Pros

  • You can embed external web content in your page, which has many applications such as:
    • Embedding third party media
    • Embedding code examples
    • Embedding third party applets like payment forms
  • All current popular browsers support iframe

Cons

  • <frames> allow to break the document into several smaller panels, which each displayed a different document. This was considered to be terribel, and the frame tag was deprecated in HTML5 but the <iframe> or inline frame is still avaliable. A pitfall is using <iframes> to mimic <frame> behavior.
  • You can embed external web content to your page, but some constructs should not be made with iframe such as photo galleries and forums

Summarizing: “If you need to embed some independent, already-existing HTML document into the current document, use an <iframe>. If you are building all of it from scratch together, there is almost no good reason to break a page design into several independent documents, especially if they aren’t actually independent pieces of content.” (http://html.com/tags/iframe/)

Question 9

CSS (in combination with HTML) allows for the separation between style markup and content markup. This separation has many benefits. It increases the accessibility of the web, e.g. screen readers are only interested in content markup and not style markup. From the programming and labor division part it is also beneficial.   1. Easier to maintain and update
  2. Greater consistency in design
  3. More formatting options
  4. Lightweight code
  5. Faster download times
  6. Search engine optimization benefits
  7. Ease of presenting different styles to different viewers

Native apps are written in one language for a single device (Java for android phones and Objective C for iOS devices) whilst web based apps live in the browser on a device and are written in Javascript and can therefore run on any device. Look up information about the new protocol for IP addresses (IPv6, IP version 6). Describe the main features of this protocol and indicate what the reasons are to move from version 4 to version 6. (Expected length: 1/4 page)

According to this Webopedia source the reason to move to IPv6 from IPv4 is from growing concern that if the number of end systems connectd to the Internet continues to grow the nuber of avaliable IP adresses will not be enough for the increased demand, so the demand for Ip adesses will exceed the supply. An IPv4 IP adress is written as four numbers from 0 to 255 separated by periods for example 1.160.10.240, whilst an IPv6 adress is written in hexadecimal and separated by colons, for example 3ffe:1900:4545:3:200:f8ff:fe21:67cf. A list of other possible benefits to IPv6 is shown below:

  • No more NAT (Network Address Translation)
  • Auto-configuration
  • No more private adress collisions
  • Better multicast routing
  • Simpler header format
  • Simplified, more efficient routing
  • True quality of service (QoS) also called flow labeling
  • Built in authentication and privacy support
  • Flexible options and extensions
  • Easier admininistration(no more DHCP)

Question 8

Briefly explain the role of events and event listeners in JavaScript.

Events and event listeners belong to the event-driven programming paradigm. Where programs are always in a never ending loop, during each iteration of the loop it is checked if any event has fired, if so all the event listeners (functions that have subscribed to the event) are called and executed.
